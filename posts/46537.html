zookeeper的安装

更新日期:   2020-11-22
文章字数:   771
阅读时长:   3 分

这里要特别感谢ysgg大佬分享他的踩坑记录，感谢他！

一、下载zookeeper

传送门

这里的版本是3.6.2

下载好后解压放在/usr/local下，并且重命名为zookeeper(不是必须)

依然接着上一篇文章说的，在hadoop用户下操作

二、配置文件

在zookeeper目录下创建文件夹dataDir、dataLogDir和bin同级

dataDir下创建三个文件夹zoo1_datadir、zoo2_datadir、zoo3_datadir,并且在每个文件夹中创建一个名为myid的文件，注意没有后缀名，内容分别为1，2，3即可

dataLogDir下创建三个目录zoo1_datalogdir、zoo2_datalogdir、zoo3_datalogdir

在conf目录下根据zoo_template.cfg复制三个文件

cp -r zoo_template.cfg zoo1.cfg
cp -r zoo_template.cfg zoo2.cfg
cp -r zoo_template.cfg zoo3.cfg

其中内容粘贴以下内容：

# The number of milliseconds of each tick
tickTime=2000
# The number of ticks that the initial 
# synchronization phase can take
initLimit=10
# The number of ticks that can pass between 
# sending a request and getting an acknowledgement
syncLimit=5
# the directory where the snapshot is stored.
# do not use /tmp for storage, /tmp here is just 
# example sakes.
#这里是要配置的缓存和日志地址
dataDir=/usr/local/zookeeper/dataDir/zoo1_datadir
dataLogDir=/usr/local/zookeeper/dataLogDir/zoo1_datalogdir
# the port at which the clients will connect
clientPort=2181
# the maximum number of client connections.
# increase this if you need to handle more clients
#maxClientCnxns=60
#
# Be sure to read the maintenance section of the 
# administrator guide before turning on autopurge.
#
# http://zookeeper.apache.org/doc/current/zookeeperAdmin.html#sc_maintenance
#
# The number of snapshots to retain in dataDir
#autopurge.snapRetainCount=3
# Purge task interval in hours
# Set to "0" to disable auto purge feature
#autopurge.purgeInterval=1

## Metrics Providers
#
# https://prometheus.io Metrics Exporter
#metricsProvider.className=org.apache.zookeeper.metrics.prometheus.PrometheusMetricsProvider
#metricsProvider.httpPort=7000
#metricsProvider.exportJvmInfo=true
#这里的server.n中的n为myid里的值
server.1=localhost:2887:3887
server.2=localhost:2888:3888
server.3=localhost:2889:3889

==注意==:
每一个文件的内容需要该的地方是

#这里是要配置的缓存和日志地址
dataDir=/usr/local/zookeeper/dataDir/zoo1_datadir
dataLogDir=/usr/local/zookeeper/dataLogDir/zoo1_datalogdir
# the port at which the clients will connect
clientPort=2181

前两个不用多说，最后一个端口号分别为2181、2182、2183

配置环境变量

在/home/hadoop/.bashrc文件后添加以下内容

export ZOOKEEPER_HOME=/usr/local/zookeeper
export PATH=$PATH:$ZOOKEEPER_HOME/bin
export ZOO_LOG_DIR=/usr/local/zookeeper/logs

配置完后记得运行命令

source /home/hadoop/.bashrc

此时配置已经完成

三、启动

运行以下命令以测试是否开启成功

zkServer.sh start zoo1.cfg
zkServer.sh start zoo2.cfg
zkServer.sh start zoo3.cfg

结果应为STARTED

运行以下命令以查看三个节点的状态

zkServer.sh status zoo1.cfg
zkServer.sh status zoo2.cfg
zkServer.sh status zoo3.cfg

结果应为follwer之类的主从关系

四、可能出现的问题

1. 开启成功，检查状态的时候发现不是主从关系

检查配置文件是否出错

重新启动zookeeper

如果还是有问题，说明zookeeper未重启成功，删除zk数据目录下之前生成的version-2文件夹，再重启。

2. 开启成功，检查状态发现开启失败

检查配置文件是否出错

重新启动zookeeper

如果还是有问题，说明zookeeper未重启成功，删除zk数据目录下之前生成的version-2文件夹，再重启。

3. 开启失败

配置文件出错
